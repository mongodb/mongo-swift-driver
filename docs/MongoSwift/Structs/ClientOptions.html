  • threadPoolSize

    MongoSwift.MongoClient provides an asynchronous API by running all blocking operations off of their originating threads in a thread pool. MongoSwiftSync.MongoClient is implemented as a wrapper of the async client which waits for each corresponding asynchronous operation to complete and then returns the result. This option specifies the size of the thread pool used by the asynchronous client, and determines the max number of concurrent operations that may be performed using a single client. Defaults to 5.
